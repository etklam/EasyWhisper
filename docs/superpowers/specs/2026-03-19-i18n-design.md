# Multi-Language UI Support Design

**Date:** 2026-03-19
**Status:** Draft

## Overview

Add internationalization (i18n) support to the FOSSWhisper desktop application, enabling users to switch between English, Traditional Chinese (zh-TW), and Simplified Chinese (zh-CN).

## Requirements

- **Languages:** English (en), Traditional Chinese (zh-TW), Simplified Chinese (zh-CN)
- **Default Language:** English (en)
- **Language Switcher:** Settings page only
- **Persistence:** User's language choice must be saved and restored across app restarts

## Tech Stack

- **Framework:** Vue 3 + Electron
- **UI Library:** Naive UI (v2.41.1)
- **i18n Library:** vue-i18n@9
- **State Management:** Pinia

## Architecture

### Directory Structure

```
apps/desktop/src/
├── locales/
│   ├── en.json         # English translations
│   ├── zh-CN.json      # Simplified Chinese
```

## Implementation Components

### 1. i18n Plugin Setup (`i18n.ts`)

```typescript
import { createI18n } from 'vue-i18n'
import en from './locales/en.json'
import zhCN from './locales/zh-CN.json'
import zhTW from './locales/zh-TW.json'

const i18n = createI18n({
  legacy: false,
  locale: 'en', // default
  fallbackLocale: 'en',
  messages: { en, 'zh-CN': zhCN, 'zh-TW': zhTW }
})

export default i18n
```

### 2. Settings Integration

- Add `language` property to existing settings store
- Persist to electron's config storage
- Load saved locale on app startup

### 3. Naive UI Locale Integration

- Wrap app with `<n-config-provider>`
- Sync Naive UI component locale with vue-i18n locale
- Import Naive UI locales: `enUS`, `zhCN`, `zhTW`

### 4. Component Refactoring

- Replace all hardcoded Chinese text with `t('key')` composable
- Organize translation keys by feature/page
- Update router meta to use translated titles


## Files to Modify

- `apps/desktop/src/i18n.ts`
- `apps/desktop/src/locales/en.json`

- All component files in `src/components/` - Replace hardcoded text
- All view files in `src/views/` - Replace hardcoded text
- `apps/desktop/src/router/index.ts` - Update meta titles
- `apps/desktop/src/stores/settings.ts` - Add language setting (or create if not exists)

```bash
pnpm add vue-i18n@9
```

## Success Criteria

1. App starts in English by default
2. Language can be changed in Settings page
3. Language preference persists across app restarts
4. All UI text displays in selected language
5. Naive UI component text (pagination, empty states, etc.) also switches languages
