# Multi-Language UI Support Design

**Date:** 2026-03-19
**Status:** Draft

## Overview

Add internationalization (i18n) support to the FOSSWhisper desktop application, enabling users to switch between English, Traditional Chinese (zh-TW), and Simplified Chinese (zh-CN).

## Requirements

- **Languages:** English (en), Traditional Chinese (zh-TW), Simplified Chinese (zh-CN)
- **Default Language:** English (en)
- **Language Switcher:** Settings page only (using NSelect dropdown)
- **Persistence:** User's language choice must be saved and restored across app restarts
- **System Locale Detection:** On first launch, detect system locale and auto-select matching language (fallback to en)

## Existing Infrastructure

The project already has partial i18n infrastructure:
- `apps/desktop/electron/main/constants/locales.ts` - Defines `VALID_LOCALES = ['en', 'zh-TW', 'zh-CN']`
- `apps/desktop/electron/main/SettingsManager.ts` - Has `locale: 'en'` default setting with validation
- Existing IPC handlers `IPC_CHANNELS.SETTINGS_GET` and `IPC_CHANNELS.SETTINGS_SET` - Support locale updates

The new implementation will integrate with these existing patterns, using `locale` as the property name (not `language`) for consistency.

## Tech Stack

- **Framework:** Vue 3 + Electron
- **UI Library:** Naive UI (v2.41.1)
- **i18n Library:** vue-i18n@9
- **State Management:** Pinia

## Implementation Components

### 1. i18n Plugin Setup (`i18n.ts`)

```typescript
import { createI18n } from 'vue-i18n'
import en from './locales/en.json'
import zhCN from './locales/zh-CN.json'
import zhTW from './locales/zh-TW.json'

const i18n = createI18n({
  legacy: false,
  locale: 'en', // default, will be overridden by saved settings
  fallbackLocale: 'en',
  missing: (locale, key) => {
    // Log missing translation keys during development
    if (import.meta.env.DEV) {
      console.warn(`[i18n] Missing translation: ${key} for locale: ${locale}`)
    }
    return key
  },
  messages: { en, 'zh-CN': zhCN, 'zh-TW': zhTW }
})

export default i18n
```

### 2. Settings Integration

**Note:** Settings are managed through IPC handlers, not a Pinia store. The existing `useWhisperStore` and `useAiStore` handle UI state.

- Use existing IPC handlers: `IPC_CHANNELS.SETTINGS_GET` and `IPC_CHANNELS.SETTINGS_SET`
- Property name is `locale` (matching existing SettingsManager.ts)
- The `toSettingsPatch` function in `apps/desktop/electron/main/ipc/settings.ts` already handles locale validation
- On app startup: Load saved locale from Electron config via IPC
- On language change: Persist via `SETTINGS_SET` IPC call

### 3. Naive UI Locale Integration

**Important:** `App.vue` already has `<n-config-provider>`. The duplicate wrapper in `HomeView.vue` should be removed.

```typescript
// Import Naive UI locales directly from 'naive-ui'
import { enUS, zhCN, zhTW } from 'naive-ui'

// Create a locale map for Naive UI
const naiveUILocales = { en: enUS, 'zh-CN': zhCN, 'zh-TW': zhTW }

// Sync with vue-i18n locale
const currentNaiveLocale = computed(() => naiveUILocales[i18n.global.locale.value])
```

### 4. Component Refactoring

**Scope:** 12+ Vue component files contain hardcoded Chinese text. Each needs individual refactoring.

- Replace all hardcoded Chinese text with `t('key')` composable from vue-i18n
- Organize translation keys by feature/page using dot notation
- Update router meta to use computed properties based on current locale
- Use existing hardcoded Chinese text as source of truth for zh-CN translations

**Components requiring refactoring:**
1. `App.vue` - Navigation header
2. `views/HomeView.vue` - Main workspace
3. `views/SettingsView.vue` - Settings page
4. `components/ImportPanel.vue` - File/URL import
5. `components/ModelSelector.vue` - Model selection

7. `components/QueueTable.vue` - Task queue table
8. `components/QueueItem.vue` - Queue item


## Testing

### Manual Testing Checklist

1. **Initial Launch**
   - [ ] Fresh install starts in English (default)
   - [ ] System locale detection works (zh-CN system → zh-CN UI, zh-TW system → zh-TW UI)

2. **Language Switching**
   - [ ] Settings page shows locale selector with all 3 options
   - [ ] Selecting a language immediately updates all UI text
   - [ ] Navigation labels update
   - [ ] All component text updates
   - [ ] Naive UI component text updates (empty states, pagination, etc.)

3. **Persistence**
   - [ ] Language preference saves to config
   - [ ] Relaunching app restores saved language
   - [ ] Survives app restarts

4. **All Languages**
   - [ ] Test each UI component in English
   - [ ] Test each UI component in Simplified Chinese
   - [ ] Test each UI component in Traditional Chinese
   - [ ] Verify no missing translation keys (check console warnings)

5. **Edge Cases**
   - [ ] Invalid locale in config falls back to 'en'
   - [ ] Missing translation key falls back to key name
   - [ ] Date/time formatting respects locale (if dates are shown)

### Development Mode

During development, missing translation keys will be logged to console:
```
[i18n] Missing translation: home.dropZone.title for locale: en
```

## Success Criteria

1. App starts in English by default (or detected system locale)
2. Language can be changed in Settings page
3. Language preference persists across app restarts
4. All UI text displays in selected language
5. Naive UI component text (pagination, empty states, etc.) also switches languages
6. No console errors related to missing translations
7. Existing Chinese UI text is preserved as zh-CN translations

## Future Considerations

### Adding New Languages

Contributors can add new languages by:
1. Adding locale to `VALID_LOCALES` in `constants/locales.ts`
2. Creating new JSON file in `src/locales/`
3. Adding import and registration in `i18n.ts`
4. Adding Naive UI locale import if available

### TypeScript Support (Optional Enhancement)

For compile-time safety, consider adding typed translation keys:
```typescript
// Define translation key schema
type TranslationKeys = {
  common: {
    back: string
    save: string
  }
  // ...
}

// Use with vue-i18n for autocomplete
declare module 'vue-i18n' {
  interface DefineLocaleMessage extends TranslationKeys {}
}
```

### Performance

All locale files will be bundled with the app. For three languages, this is acceptable (~10-20KB total). If language count grows significantly, consider lazy-loading:
```typescript
const messages = {
  en: () => import('./locales/en.json'),
  'zh-CN': () => import('./locales/zh-CN.json'),
  'zh-TW': () => import('./locales/zh-TW.json'),
}
```
